# Release Management Policy

## Purpose

Provide a controlled process for deploying software changes to production.

---

## Release Types

### Major Release

Examples:

- New Service
- New Architecture
- New Payment Flow

Approval Required:

- CTO
- Chief Architect

---

### Minor Release

Examples:

- Feature Enhancement
- API Extension

Approval Required:

- Engineering Manager

---

### Patch Release

Examples:

- Bug Fix
- Security Fix

Approval Required:

- Team Lead

---

## Deployment Windows

Production Deployments:

Saturday

10:00 PM to 02:00 AM

---

## Mandatory Requirements

- Testing Complete
- Security Scan Passed
- PRR Approved
- Rollback Plan Available
